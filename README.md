# 📊 Customer Shopping Behavior Analysis
---

## 🔍 Overview
This project analyzes customer shopping behavior using transactional data to uncover insights into:
- Spending patterns  
- Customer segments  
- Product preferences  
- Subscription impact  

It demonstrates an **end-to-end data analytics workflow**:
➡️ Python (EDA & Cleaning) → SQL (Analysis) → Power BI (Dashboard)

## 📁 Dataset
- **Records:** 3,900 transactions  
- **Features:** 18 columns  

### Key Attributes:
- 👤 Customer: Age, Gender, Location, Subscription  
- 🛒 Purchase: Product, Category, Amount, Season  
- 📊 Behavior: Discount, Ratings, Frequency, Shipping  

---

## 🛠️ Tech Stack
| Tool | Purpose |
|------|--------|
| 🐍 Python (Pandas, NumPy) | Data Cleaning & EDA |
| 🗄️ MS SQL Server | Data Analysis |
| 📊 Power BI | Dashboard & Visualization |
| 📄 CSV / Excel | Data Storage |

---

## ⚙️ Project Workflow

### 🔹 1. Data Cleaning (Python)
- Handled missing values (Review Ratings)  
- Standardized column names  
- Feature Engineering:
  - `age_group`  
  - `purchase_frequency_days`  
- Removed redundant columns  

---

### 🔹 2. SQL Analysis
Key business queries:
- Revenue by gender  
- Top-rated products  
- Subscriber vs non-subscriber analysis  
- Customer segmentation (New, Returning, Loyal)  
- Revenue by age group  

---

### 🔹 3. Dashboard (Power BI)
Interactive visuals:
- 📈 Revenue trends  
- 👥 Customer segments  
- 🛍️ Product performance  
- 🚚 Shipping insights  

---

## 📈 Key Insights
✔️ Subscribers contribute higher revenue  
✔️ Repeat customers have strong lifetime value  
✔️ Some products perform well without discounts  
✔️ High-value age groups drive sales  
✔️ Express shipping users spend more
